# 憲法批准チェックリスト（判断②・H1）

> `PHILOSOPHY-venture-os-draft-v1.md`（V1〜V6・草案）を VO 独立 repo の正式 `philosophy.md`（L-FROZEN-PHIL）へ昇格する手順。**H1（哲学変更）は人間専管**。AI は昇格の準備（本書）まで。

## 批准とは何か（可逆→不可逆の境界を跨ぐ）

草案（draft）は可逆に編集できる。批准すると V1〜V6 は **L-FROZEN-PHIL**（AI 提案 PR も不可・第9条）になり、以降の改訂は人間専管＋ゲート後再諮問を要する。批准は「止める基準そのものを固定する」行為ゆえ H1。

## 批准前の確認事項（人間が最終判断）

- [ ] V1〜V6 の条文が意図どおり（信用一次・充足・代謝ループ・継続不変条件・真正性・自律境界）
- [ ] VTL と「信用資本」の峻別が保たれている（内部信用 vs 外部信用）
- [ ] DH 9条との接続表に矛盾がない（V1〜V6 は 9条の責務再定義でなく事業素材への適用明文化）
- [ ] 未決だった信用側3穴（戦略#0.5・測定#2・裁定#3）が埋まり、V1/V4 が運用可能な一貫線になっている
- [ ] 条番号・本数（V1〜V6）を確定（統合/分割の要否）

## 批准手順（決定後）

```
1. PHILOSOPHY-venture-os-draft-v1.md の "草案" ステータスを外す
2. ファイル名 → philosophy.md（VO repo ルート）
3. 前文に「下層 = 継承した DH 9条（継承方式は判断①）／上層 = 本 VO V1〜V6」を明記
4. frontmatter/冒頭に status: ratified / L-FROZEN-PHIL / ratified_at / ratified_by(human) を記録
5. 委譲境界 SPEC（delegation-boundary 相当）に VO 版を配置（自己改訂禁止・L-FROZEN-META）
6. `history/PHILOSOPHY-CHANGELOG.md`（VO repo・無ければ新規作成）に批准エントリを追記（日付・批准者・V1〜V6 確定の記録）
```

## 推奨タイミング

**(a) 草案のまま立ち上げ → repo で批准**（可逆・流速優先）。独立 repo 立ち上げ（判断①）と同期して批准するのが自然。それまでは草案として可逆に扱い、A で埋めた信用側3穴の運用感を design_only 机上で確かめてから固定できる。

## 批准後に変わること

- V1〜V6 は AI が提案 PR すら作れない（L-FROZEN-PHIL）。改訂は人間専管＋Council 再諮問。
- VO の全スキルは批准済み philosophy.md を原典参照（草案参照からの切替）。
- 「憲法の自己改訂禁止」が VO でも発効（VO は自らの委譲境界 SPEC を自ら改訂できない）。
