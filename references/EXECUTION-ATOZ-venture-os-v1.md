# 事業執行の A to Z ── venture-os カバレッジマップ v1

> 目的: 事業執行（venture execution）の全域を洗い出し、VO が **今どこまでカバーし、何が穴（GAP）か** を可視化する抜け漏れ検査。単なる汎用チェックリストではなく、各要素を **責務（層）／憲法条／venture_scope／現状カバレッジ** に紐づける。
> 背骨: 憲法 V3 の信用⇄貨幣 代謝ループ。全フェーズはこのループの一部として位置づく。
> 参照: `PHILOSOPHY-venture-os-draft-v1.md`（V1〜V6）/ `CONCEPTION-venture-os-metaskill-v2.md` / `prototype/bizmodel-pattern-library.md` / `CREDIT-STRATEGY-venture-os-v1.md`

## 凡例

- **責務**: L0=設計責務 / L1=実働責務 / Rev=独立市場検証者 / Council=判断機構 / L2=統括（休眠）/ H=人間専管
- **scope**: DO=design_only / GTM=gtm_execution / OPS=full_ops（憲法 V6・第9条で段階解放）
- **カバレッジ**: ✅covered（**設計成果物に記載済み** ＝ 設計レベルで確定。実装/実体生成を意味しない）/ 🟡partial（触れているが未詳細）/ ❌gap（未着手・要設計）/ ⏸deferred（scope 未到達）/ ➡delegated（DH/人間/外部ツール）

## 全体マップ（代謝ループ V3 を背骨に）

```
 [Phase 0 起点]──設計→ [Phase 1 検証]──実証→ [Phase 2 構築]──提供→ [Phase 3 獲得]
   L0 経営責務            L1+Rev              L1(+DH連携)          L1 信用獲得↑
      ▲                                                              │
      │ (V4: 対応は層でなく L0 へ差し戻す)                            ▼
 [Phase 7 拡大/転換/終結] ←継続─ [Phase 6 運営] ←充足─ [Phase 5 財務] ←マネタイズ─ [Phase 4 収益化]
   L0+H 人間専管           二runway監視         貨幣runway/不可逆ゲート        L1+価格ガバナンス
                                    │
                    ┌───────────────┴───────────────┐
              [横断 A ガバナンス]  [横断 B 法務]  [横断 C 測定]  [横断 D 学習・代謝]
```

信用（stock）は左回りに蓄積し、貨幣（flow）は Phase 4-5 で「継続に足るだけ」代謝され（V2 充足）、Phase 6 で二 runway を監視し、枯渇・毀損は Phase 0（L0）へ新テーゼとして戻る（V4・L3 非新設）。

| 優先 | GAP | なぜ重要か（憲法接続） | 埋め方の方向 |
|---|---|---|---|
| ~~★★★~~ ✅**済** | ~~0.5 信用戦略の設計手法~~ | V1 が「信用が一次目的」と定めたのに「どう張るか」の方法が空だった | **`CREDIT-STRATEGY-venture-os-v1.md` で解決**（信用ポジション設計・CQ1〜CQ4・§9 で測定への橋を敷設） |
| ~~★★★~~ ✅**済** | ~~6.1/C.1 信用資本の測定（信用runway）~~ | V4 の二軸監視の片肺だった | **`CREDIT-RUNWAY-venture-os-v1.md` で解決**（代理指標束→合成スコア→危険水準までの期数。貨幣runway と同一単位化。`vo-crosscut-credit-runway` スキル） |
| ~~★★★~~ ✅**済** | ~~二runway 競合裁定~~ | 信用のための赤字 vs 黒字化の信用犠牲の原則が無かった | **`TWO-RUNWAY-ARBITRATION-venture-os-v1.md` で解決**（生存下限は貨幣優先・下限外は信用優先・種別調整・Council two-runway プロトコル・構造的競合は pivot） |
| ★★ | **3.4 信用獲得の実働／3.5 CS** | V1 の一次目的を実際に回す実働が空 | L1 に信用獲得プレイブック＋CS プロトコル |

## VO がやらない／委譲する境界（誤って抱え込まない）

- **➡ DH へ**: プロダクト実装・セキュリティ/信頼性の技術検証（2.2/2.5）。VO は THESIS を渡すところまで。
- **➡ 人間専管（H・第9条 L-FROZEN）**: 資金調達・撤退・EXIT・雇用・重要契約・知財/規制の最終判断（5.2/7.3/7.4/D.2/B系）。継続を賭ける不可逆行為は自律しない（V6）。
- **➡ 外部ツール**: 会計・税務（freee 等）、決済（Stripe 等）。VO はセンサー入力として読むだけ。
- **層にしない**: 対応・運営（6.3）は常設層でなく L0 差し戻し（L3 非新設・V4）。

## 一行

**事業執行の A to Z は「信用⇄貨幣 代謝ループ（V3）の7フェーズ＋4横断」で漏れなく畳める。骨格は既にあり、埋めるべき穴は主に "信用側"（信用の張り方・測定・二runway裁定・獲得実働・毀損対応）と、不可逆リスク（知財/規制/回収）と外部連携（会計）に集中している。**
