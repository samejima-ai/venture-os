## 1. 事業執行の5層検証スタック（DH 運用時検出塔の事業版）

**左ほど優先・安い・決定論的。下層は上層が完遂してから発動。推論・人間は最後の砦。** DH の 30/30/20/10/7/3 配分思想を継承。

**土地（Shift Left 基盤・設計時予防・スタック外）**: `vo-crosscut-unit-economics` の算数で貨幣破綻を事前排除。運用に入る前に常時成立している前提（第2条）。基盤 FAIL＝「設計で防げた破綻が運用に流出」＝メタ FAIL として独立集計。

| 層 | 事業版の名称 | 何を検証するか | 種別 | 事業での ground truth の例 |
|---|---|---|---|---|
| **第1層** | 計算センサー | 数字が閉じるか（コスト・粗利・支出上限） | 決定論・極小 | 実コストの再計算／支出が `monetary_gate` 内／粗利 = 売上 −（推論＋人間検証）が正 |
| **第2層** | 成果検証（E2E相当）★ | 主張した成果が**実際に起きたか** | 決定論・小 | 状態 write 照合 ── 納品物が存在し読める／チケット status=resolved／コード compile+test green |
| **第3層** | コスト測定 | 人間検証コスト vs 粗利（採算） | 定量・小 | 検証工数を COGS 計上し、限界費用が売上を割らないか |
| **第4層** | 推論/rubric センサー | 決定論で測れない品質（text/content） | 確率的・中 | rubric チェック＋LLM 評価（**過信禁止**・LLM 判定は AUROC ≤0.65） |
| **第5層** | 人間 QA / 献上 | 不可逆・信用・真正性 | 人間・大 | 対外公開・成果確定・不可逆判断・信用毀損の兆候 |

**発動順序の不変条件**: 第1層（数字）が通る前に第2層（成果）を回さない。第2層（決定論）が通る前に第4層（推論）を使わない。**「型・テスト通過してから推論検証」の事業版＝「算数と成果照合が通ってから、品質の主観評価や人間判断」**。左で解けるものを右で解かない（推論・人間は希少資源）。

## 2. 独立検証の実装規格（false success 対策・DH independent-reviewer の事業版）

AI は自己申告の"成功"を信用できない（失敗の44-90%が false success）。防御は**独立検証**のみ。DH の independent-reviewer の設計原則を事業執行へ翻訳:

| DH の原則 | VO:L1 での規格 |
|---|---|
| **実装コンテキスト隔離**（自己検証バイアス排除） | 検証は実行エージェントの**生成文脈・試行錯誤ログ・自己申告を入力にしない**。入力は「成果物＋THESIS/DONT-VENTURE/受入基準」のみ |
| **agent 本体はプロジェクト不変・差異は入力に閉じる** | 検証ロジックは事業不変。事業差異は入力（成果物・ground truth 定義・受入チェックリスト）に閉じる |
| **一方向依存**（harness-verifier→本体、逆禁止） | 検証系は実行系に依存しない。実行系の変更で検証が壊れない。依存は「検証→読み取り→状態」の一方向 |

### 変換種別ごとの ground truth と判定

| 変換 | ground truth（何で PASS を決めるか） | 検証層 |
|---|---|---|
| **code/program** | compile 通る・test green・diff が主張と一致 | 第2層（決定論・DH harness-verifier / autonomous-dev の自己検証に委譲可） |
| **agent/workflow** | 外部状態が実際に変わった（DB 行・チケット resolved・ファイル生成＆読める） | 第2層（状態照合） |
| **text/文章・content** | プログラム的 ground truth が**無い** → rubric＋人間 QA 必須（自動 PASS 禁止） | 第4層→第5層 |
| **image/audio/video** | **未検証モダリティ ── INACTIVE**（`EXECUTION-BASIS` §7）→ 献上 | 第5層（人間） |

### PASS / FAIL / 再試行 / 献上のルール

```
AI が「成功」と報告
  → 該当層で検証を実行（自己申告は無視）
  → PASS: 成果として計上
  → FAIL & 1回目: 一度だけ再試行
  → FAIL & 2回目: 成功と報告しない・献上（人間へ）
  → 検証手段が無い変換（text 等）: 自動 PASS 禁止・第5層（人間 QA）へ強制
  → dual-control が可能なら独立 verifier を立てる（false success を ~15x 抑制）
```

## 3. 各荷重要素の行動指針（状況 → 行動）

| # | 状況 | 行動 |
|---|---|---|
| L1-2 | タスクの想定時間 > 1時間 | 検証可能なチェックポイントに分割（一気に走らせない・第2層で各点照合） |
| L1-1 | AI が成功と言うが検証 FAIL | 成功扱いしない・1回再試行・ダメなら献上（§2 のルール） |
| L1-1 | 変換に決定論的 ground truth が無い（text 等） | 自動 PASS 禁止・第4層 rubric→第5層 人間 QA へ |
| 2 | 変換が image/audio/video | INACTIVE・人間へ献上（未検証モダリティ） |
| L1-5 | 市場が B2C 真正性型（作品性・人間性が価値） | AI を隠さない・人間を前面に or 回避（V5・真正性を偽らない） |
| L1-5 | 市場が B2B 成果検証可能 | 主戦場として進める（provenance 非依存・検証可能） |
| L1-4 | 不可逆行為（契約/資金/雇用/撤退/対外公開/価格恒久変更）の直前 | **停止・人間専管**（L-FROZEN・V6）。VTL がいくら高くても委譲しない |
| L1-3 | 人間検証コスト > 粗利 | そのユニットは成立しない・pivot 献上（充足しない） |
| L1-6 | 自己申告の生産性 vs 実測が乖離 | **実測を採用**（自己申告は+20%誤認する）。earned のみ計上 |
| 土地 | 第1層で数字が破綻（LTV/CAC<1・支出超過） | 実行を止め、L0（THESIS）へ差し戻し（設計で潰す） |
| 献上 | 信用毀損の兆候（苦情・解約急増・炎上） | L0 へ差し戻し（新テーゼの発見・L3 非新設・V4） |

## 4. 献上トリガー一覧（検知したら人間へ上げる）

第6条（人間最終承認）に基づき、次を検知したら実行を止めて献上する:

1. 検証 FAIL が再試行後も解消しない（false success の疑い）
2. 決定論的 ground truth の無い変換の**対外確定**（text/content の公開・納品）
3. 不可逆枠（金・法・人・撤退・調達・対外恒久変更・規制産業判断）
4. 充足水準を割る採算（人間検証コスト > 粗利）
5. 未検証モダリティ（image/audio/video）
6. 信用毀損の兆候（V5・earned が負に転じる予兆）
7. 長ホライズン（>1時間相当）の非分割タスク

## 5. DH 機構への対応（研鑽の出所・トレーサビリティ）

| VO:L1 の要素 | 翻訳元（DH 実装済み機構） |
|---|---|
| 5層検証スタック（§1） | DH「5層エラー検出スタック」（inferential-sensor-v2・30/30/20/10/7/3・左優先・下層は上層後） |
| 独立検証の文脈隔離（§2） | DH `layer1-independent-reviewer`（実装コンテキスト隔離・自己検証バイアス排除・入力はドキュメントと成果物のみ） |
| 一方向独立（§2） | DH `harness-verifier`（本体に依存しない・依存は検証→読み取りの一方向・自己検証の盲点への解） |
| 献上（§4） | DH 第6条 人間最終承認・献上哲学 |
| 土地（Shift Left 基盤） | DH 第2条 Shift Left（設計時予防・運用スタック外・基盤 FAIL は独立集計） |

**要点**: VO:L1 は検証機構を新規発明していない。DH が「ソフト生成物の false success」に対して実装した独立検証 DNA を、「**事業成果の false success**」へ翻訳しただけ。DH で battle-tested な構造を継承する（第3条・再実装しない）。
