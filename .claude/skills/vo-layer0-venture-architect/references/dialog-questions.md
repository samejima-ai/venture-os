# VO:L0 対話質問集 ── 非専門家から事業テーゼを引き出す

> SKILL.md ステップ2の運用参照。DH `dialog-questions.md` の事業版。人間が「そうそう/違う」で判定できる具体的な問いを、選択肢付きで投げる。抽象質問（「どんな事業ですか？」）は避ける。1応答で詰め込みすぎない。

## ステップ2: 顧客 × 課題 × 価値提案（機能×条件の事業版）

| 狙い | 問い（選択肢提示の例） |
|---|---|
| 対象顧客 | 「最初のお客さんは誰に近いですか？ (a) 個人消費者 (b) 中小事業者 (c) 大企業 (d) 開発者 (e) クリエイター」 |
| 課題 | 「その人の"今お金や時間を無駄にしている瞬間"は？ 具体的な場面で」 |
| 価値提案 | 「あなたの解き方は既存のどれに近い？ (a) 既存より安い (b) 既存より速い/楽 (c) 既存に無い体験 (d) 既存が無視する層向け」 |
| 代替 | 「今それを人はどう間に合わせている？（競合・自作・我慢のどれ）」 |

到達目安: 「<顧客> の <場面> の <痛み> を <手段> で解く」が1文で書ける粒度。

## モード判定情報（並行取得・VENTURE-REGIME 用）

| 軸 | 問い |
|---|---|
| 事業ステージ | 「今どこ？ (a) アイデアだけ (b) 作る対象は決まった (c) お客がつき始めた (d) 拡大したい」 |
| 不可逆コスト | 「立ち上げに前借り（借入・法人・雇用・在庫）は要りますか？ 要るなら何が」 |
| ソフト要否 | 「提供にソフト/アプリの新規開発は要りますか？（要れば後で DH に発注）」 |
| 充足水準 | 「"これだけ稼げれば続けられる"月額はいくらくらい？（最大化でなく継続ライン）」 |

## 信用戦略（CQ1〜4）の噛み砕き（credit-strategy §8 の対話例）

- CQ1 源: 「一番最初に"信頼された"と言えたら嬉しい相手は誰？」
- CQ2 種別: 「その人に何を一番信じてほしい？ ①ちゃんと動く実力 ②絶対に嘘つかない誠実さ ③困った時に親身な関係」
- CQ3 経路: 「その信頼、何で伝える？ ①実績・数字 ②正直な価格と規約 ③コミュニティ・サポート ④第三者の認証」
- CQ4 代謝: 「その信頼を、信頼を損なわずにどうお金に換える？（例: 実力信頼→成果課金、関係信頼→会員、誠実信頼→データ同意）」

## 未回答時の自動補完（確度: AI推定 を付す）

- 対象顧客: 価値提案の直接受益者を推定
- 信用種別: 提供価値の性質から推定（インフラ→能力、扱いが繊細→誠実、継続関係→関係）
- 充足水準: 業界水準の最小運営費を仮置き
- 未回答は THESIS（草案）/INTENT から推定し、献上時に人間が確認する（自律駆動を止めない・第4条）
