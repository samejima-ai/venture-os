## eval 4: 貨幣は黒字だが核代謝が信用を焼く（ダークパターン解約）

**狙い**: money-viable（LTV/CAC 良好）だが核代謝が誠実信用を不可避に焼く事業を、L0 が捕まえられるか。V1（信用一次）の実装度テスト。

### 初回トレース（修正前） — ⚠ 穴を検出

| assertion | 判定 | トレース |
|---|---|---|
| 核代謝が一次信用を焼くなら受け入れ基準を通さない | ❌ | **受け入れ基準#2 は LTV/CAC≥1 のみを門番**。ダークパターン解約は数字が良いので #2 を通過してしまう。信用側の門番が無い |
| 信用破綻が pivot 献上に至る | ❌ | pivot 献上経路は貨幣破綻（タイプA）専用。信用破綻に対称経路が無い |
| ダークパターン解約が DONT-VENTURE | ⚠ | ステップ3「信用を焼く代謝は DONT へ」で拾えるが、**核代謝そのもの**を DONT に移すと事業が空になる矛盾（DONT に落とすだけでは不十分） |

**根本原因（非対称）**: VO の主軸は V1（信用一次）なのに、self-check（9.4）と受け入れ基準・pivot 経路が**貨幣側だけを門番**していた。信用破綻には対称ガードが無い。

### 修正（SKILL.md v0.2.0）

- フローに **5.5 信用破綻チェック（V1 対称ガード）** を新設。核代謝が一次信用を不可避に焼くなら貨幣黒字でも V1 破綻＝ pivot 献上（信用破綻型）。回避可能な副次活動のみ DONT-VENTURE へ。
- 受け入れ基準#2 を **貨幣・信用の両 viability（V4 二軸）** に拡張。
- セクション `### 5.5` を追加し、信用側尺度を `CREDIT-RUNWAY`、拮抗時の裁定を `TWO-RUNWAY-ARBITRATION`（Council）へ接続。

### 再トレース（修正後） — ✅

| assertion | 判定 |
|---|---|
| 核代謝が一次信用を焼くなら受け入れ基準を通さない | ✅ 受け入れ基準#2(b) で門番 |
| 信用破綻が pivot 献上に至る | ✅ ステップ5.5 で信用破綻型 pivot |
| ダークパターン解約が DONT-VENTURE | ✅ 副次なら DONT／核代謝なら pivot の切り分けが明示 |
